# Исправление ошибок Tailwind CSS в глобальных стилях

## Метаданные
* **Тип:** bug
* **Приоритет:** high

* **Дата завершения:** 2024-07-15

## Описание

При запуске проекта возникали ошибки Tailwind CSS связанные с неизвестными utility-классами:
```
Error: Cannot apply unknown utility class: border-border
Error: Cannot apply unknown utility class: bg-background
```

Проблема связана с использованием директивы `@apply` с пользовательскими классами, которые не определены напрямую в Tailwind.

## Требования

1. Исправить ошибки компиляции Tailwind CSS
2. Обеспечить корректную работу глобальных стилей
3. Обновить документацию с описанием правильного использования CSS-переменных в проекте

## План реализации

1. Исследовать ошибки связанные с неизвестными utility-классами
2. Заменить использование `@apply border-border` и `@apply bg-background` на нативные CSS-свойства с использованием CSS-переменных
3. Проверить работоспособность стилей
4. Обновить документацию с рекомендациями по использованию стилей

## Решение

Проблема была решена путем замены директив `@apply` на стандартные CSS-свойства:

1. `@apply border-border` → `border-color: hsl(var(--border))`
2. `@apply bg-background text-foreground` → `background-color: hsl(var(--background)); color: hsl(var(--foreground))`
3. Все остальные использования `@apply` с нестандартными классами также были заменены на нативный CSS

После замены на нативный CSS все ошибки исчезли и приложение заработало корректно.

## Выводы и рекомендации

1. Избегать использования `@apply` с нестандартными классами, не определенными в Tailwind.
2. При работе с CSS-переменными определять цвета через HSL: `hsl(var(--color-name))`.
3. В tailwind.config.js следует использовать формат `hsl(var(--color-name))` для определения цветов.
4. Для компонентов с комплексными стилями лучше использовать обычный CSS внутри `@layer components` без директив `@apply`.
