# Задача: Настройка Strapi как CMS для SuperLanding

**Тип:** feature
**Приоритет:** high
**Статус:** backlog
**Дата создания:** 2023-12-02

## Описание
Необходимо настроить базовую инсталляцию Strapi как headless CMS для проекта SuperLanding. Strapi будет использоваться как основное хранилище контента для лендингов и блога, с поддержкой локализации.

## Технические требования
- Установка и конфигурация Strapi v4/v5
- Создание основных типов контента (Content Types)
- Настройка API доступа для фронтенда
- Обеспечение поддержки локализации контента
- Создание базовых компонентов для Dynamic Zones

## План реализации
1. **Этап 1: Инсталляция и базовая настройка**
   - Создать новый проект Strapi
   - Настроить базу данных (PostgreSQL предпочтительно)
   - Настроить базовые переменные окружения
   - Проверить доступность админ-панели
   
2. **Этап 2: Создание типов контента**
   - Создать тип контента LandingPage со следующими полями:
     - slug (string, unique)
     - title (string, локализуемый)
     - metaDescription (text, локализуемый)
     - contentBlocks (Dynamic Zone)
     - SEO-поля (canonicalUrl, openGraphImage)
   - Создать тип контента BlogPost со следующими полями:
     - slug (string, unique)
     - title (string, локализуемый)
     - publishDate (datetime)
     - excerpt (text, локализуемый)
     - content (Rich Text, локализуемый)
     - featuredImage (media)
     - metaDescription (text, локализуемый)
     - SEO-поля
   
3. **Этап 3: Создание компонентов**
   - Создать компоненты для Dynamic Zones:
     - Hero (title, subtitle, ctaButtonText, ctaButtonLink, backgroundImage)
     - FeatureList (заголовок + набор FeatureItem)
     - FeatureItem (icon, title, description)
     - Testimonial (quote, authorName, authorTitle, authorImage)
     - CTA (title, text, buttonText, buttonLink)
     - RichText (стандартный редактор форматированного текста)
   
4. **Этап 4: Настройка API и локализации**
   - Настроить плагин i18n для локализации
   - Включить поддержку для английского и русского языков (минимум)
   - Создать API токены для доступа со стороны фронтенда
   - Настроить API маршруты для получения лендингов и статей блога с локализацией
   
5. **Этап 5: Тестирование и документация**
   - Создать тестовые данные (минимум один лендинг и одну статью блога)
   - Проверить получение данных через API
   - Документировать структуру API и типы данных
   - Документировать процесс добавления новых типов контента

## Критерии готовности
- [ ] Strapi установлен и доступен для разработки
- [ ] Созданы и настроены все необходимые типы контента
- [ ] API настроен и возвращает данные в ожидаемом формате
- [ ] Локализация настроена и работает корректно
- [ ] Тестовые данные созданы и доступны через API
- [ ] Документация API и процессов создана
